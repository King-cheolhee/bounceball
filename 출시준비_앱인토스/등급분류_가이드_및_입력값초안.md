# 탱탱볼해금 — 게임 등급분류 가이드 & 콘솔 입력값 초안

> 작성일: 2026-06-14 · 근거: `앱인토스_게임개발_가이드_정리.md` §3·§13, 게임 코드(story.ts·skins.ts·stages.ts)
> 목적: **출시 일정의 가장 긴 병목(1~2주)인 게임 등급분류**를, 비개발자도 그대로 따라 신청할 수 있게 정리.

---

## 0. 가장 먼저 — 절대 틀리면 안 되는 한 가지 🚨

**게임물 "등록자명"과 앱인토스 "사업자명"이 다르면 검수에서 무조건 반려됩니다.**
- 등급분류를 받을 때 쓰는 이름(구글: 개발자명 / 애플: 제공자명 / GRAC: 신청자명)을
- 앱인토스 콘솔에 등록한 **사업자(법인/개인사업자) 이름과 똑같이** 맞추세요.
- 본인은 사업자등록을 보유 중이므로, **등급분류 신청도 그 사업자 명의로** 진행하면 됩니다.

---

## 1. 우리 게임은 어떤 등급인가 (먼저 알고 시작)

탱탱볼해금은 **추상 픽셀 플랫포머**입니다. 등급에 영향을 주는 요소를 코드 기준으로 점검하면:

| 항목 | 우리 게임 | 근거 |
|------|----------|------|
| 폭력성 | 없음 (캐릭터·유혈 없음, 죽음은 픽셀이 사라지는 추상 연출) | `story.ts`, `stages.ts` |
| 선정성 | 없음 | 전체 흑백 픽셀 |
| 공포 | 없음 | — |
| 언어(욕설) | 없음 | 텍스트 최소·건조 |
| 약물·범죄 | 없음 | — |
| **사행성(도박)** | **없음** (가챠·랜덤박스 없음, 스킨은 인게임 부품 ◆로 고정가 구매) | `skins.ts` cost=20/45/75 (현금 아님) |
| **현금 인앱결제** | **없음** | 결제 코드 전무 |
| **광고** | **있음** (전체화면 광고) | `ads.ts` |
| 온라인 상호작용 | 없음 (멀티·채팅 없음, 리더보드 1차 제외) | — |

➡️ **예상 이용등급: 「전체이용가」(3세 이상 / ALL)**, 내용정보 표시사항 없음, 단 "광고 포함"만 해당.

> 이 게임은 §13 "웹보드 게임"(포커·고스톱 등 도박형)이 **아니므로**, "IARC 국내 효력 없음" 규정은 우리와 무관합니다. 아래 두 방법 모두 우리에게 유효합니다.

## 2. 두 가지 방법 비교 (핵심 의사결정)

앱인토스 콘솔이 등급분류를 인정하는 길은 둘입니다.

| 구분 | **방법 1: 오픈마켓 자체등급분류** | **방법 2: 게임물관리위원회(GRAC) 직접심의** |
|------|----------------------------------|---------------------------------------------|
| 한 줄 | 구글플레이/앱스토어/원스토어 등에 **게임을 따로 올려** 자동 등급을 받음 | GRAC에 **직접 신청**해 등급분류증명서(PDF)를 받음 |
| 콘솔 제출물 | 스토어 URL, 등록자명, 자체등급분류사업자명, 등급분류일자·번호, 이용등급·내용정보, 대표자 인감/사인, 플레이화면 4장 | 등급분류증명서 PDF 1개 |
| 등급 받는 속도 | 구글플레이 설문 기반은 보통 **즉시~수시간** | 신청 후 **수일~2주**(플랫폼·물량에 따라) |
| 비용 | 구글 개발자 등록 **1회 $25**(앱스토어는 연 $99) | GRAC **심의 수수료 별도**(금액은 GRAC 공지 확인 필요) |
| 숨은 비용/조건 | ⚠️ **게임을 그 마켓에도 실제로 출시(또는 등재)해야 함** — 지금은 앱인토스용 웹/Granite 미니앱이라, 안드로이드 앱(APK/AAB)으로 따로 빌드·등록하는 추가 작업 필요 | 다른 마켓에 안 올려도 됨. 대신 신청서·게임 설명·플레이 영상/빌드 제출 등 서류 절차 |
| 어울리는 경우 | 어차피 **구글플레이에도 배포할 계획**이면 일석이조 | **앱인토스에만** 낼 거면 더 곧장 |

> ⚠️ 수수료·소요기간의 정확한 수치는 시기에 따라 달라집니다. **GRAC(grac.or.kr)·구글플레이 콘솔의 현재 공지를 최종 확인**하세요. (이 문서는 추측값을 단정하지 않습니다.)

### 추천
- **앱인토스 단독 출시가 목표라면 → 방법 2(GRAC 직접심의)** 가 군더더기 없습니다(다른 스토어용 앱을 또 만들 필요 없음).
- **이왕이면 구글플레이에도 깔리게 하고 싶다 → 방법 1**. 단, 안드로이드 앱 별도 빌드/스토어 등록이라는 **추가 프로젝트**가 생긴다는 점을 감안하세요.

➡️ **둘 중 무엇으로 갈지만 정해 주시면**, 그 방법의 단계별 신청을 더 자세히 도와드리겠습니다.

## 3. 방법 2(GRAC 직접심의) 진행 순서 — 앱인토스 단독 출시 기준

> 가장 곧은 길이라 기본 추천. 정확한 화면·수수료는 GRAC 사이트에서 확인하며 진행.

1. **GRAC 회원가입·신청자 등록** — 본인 사업자 명의로. (등록자명=사업자명 일치 필수)
2. **게임물 등급분류 신청서 작성** — 플랫폼: 모바일(오픈마켓/그 외), 게임명, 장르(액션/캐주얼), 제작·배급사 정보.
3. **게임 내용 제출** — 게임 설명서, 플레이 영상 또는 플레이 가능한 빌드, 스크린샷.
   - ⚠️ 이 자료는 **최종 게임 화면**이 필요 → 진행 중인 S11~S20 튜닝 마무리 후가 좋음(아래 4번 참고).
4. **수수료 납부 → 심의 → 등급분류증명서(PDF) 수령**.
5. 앱인토스 콘솔 「게임 등급분류」에 **PDF 첨부**.

## 3-2. 방법 1(오픈마켓 자체등급분류) 진행 순서 — 구글플레이 예시

1. 구글플레이 **개발자 계정 등록**($25, 사업자 명의).
2. 안드로이드 앱(AAB) 빌드 — *현재 앱인토스용 미니앱과 별개의 빌드가 필요(추가 작업).*
3. 플레이 콘솔에서 **콘텐츠 등급 설문(IARC)** 작성 → 자동으로 **전체이용가** 산출.
4. 앱 등재(스토어 페이지 생성) → **스토어 URL·등급분류번호·일자** 확보.
5. 앱인토스 콘솔에 아래 4번의 값들을 입력.

## 4. 앱인토스 콘솔 「게임 등급분류」 입력값 초안

> 콘솔에 그대로 옮겨 적을 값입니다. `〔 〕` 는 신청 결과가 나오면 채울 칸입니다.

| 콘솔 입력칸 | 입력값 (우리 게임) |
|------------|--------------------|
| 이용등급 | **전체이용가** |
| 내용정보(선정·폭력·공포·언어·약물·범죄·사행) | **모두 해당 없음** |
| 광고 포함 여부 | **포함** (전체화면 광고) |
| 현금 인앱결제 | **없음** |
| 등록자명 | 〔본인 **사업자명과 동일**하게〕 |
| 자체등급분류사업자명 *(방법1만)* | 〔등급분류 결과 화면에 표시된 사업자명 그대로 — 예: 구글플레이면 그 표기〕 |
| 등급분류일자 *(방법1만)* | 〔스토어 등급 산출일〕 |
| 등급분류번호 *(방법1만)* | 〔스토어/증명서 번호〕 |
| 스토어 게임페이지 URL *(방법1만)* | 〔구글플레이/앱스토어 주소〕 |
| 등급분류증명서 PDF *(방법2만)* | 〔GRAC 발급 PDF 첨부〕 |
| 대표자 인감/사인 이미지 *(방법1)* | 〔사업자 대표 인감 또는 사인 스캔〕 |
| 플레이 화면 | 〔방법1: 자체등급분류 게임 2장 + 앱인토스 게임 2장 / 방법2: 증명서 기준〕 — 캡처는 제가 헤드리스로 규격 맞춰 뽑아드릴 수 있음 |

### 참고: IARC 설문(방법1) 핵심 답변 가이드
- 폭력/공포/성적표현/욕설/도박/약물 → **전부 "없음/아니오"**
- 사용자간 상호작용·위치공유·개인정보 → **없음**
- 디지털 구매(현금) → **없음** / 광고 표시 → **있음**
- 결과: **전체이용가(ALL)** 로 산출되는 것이 정상.

## 5. 자주 막히는 부분

- **이름 불일치 반려**: §0 참고 — 등록자명=사업자명. 가장 흔한 반려 사유.
- **최종 화면 필요**: 등급분류 제출 자료와 스토어 스크린샷 모두 "완성된 게임 화면"이 필요. 그래서 S11~S20 튜닝·스테이지 잠금(완료됨)을 끝낸 뒤 캡처하는 것이 효율적.
- **수수료/소요 추정 금지**: GRAC·구글 공지의 현재 값으로 확인. 이 문서는 단정하지 않음.
- **방법1의 함정**: "구글플레이에 자동 등급 받으면 끝"이 아니라, **안드로이드 앱을 실제로 빌드·등재**해야 그 결과(URL·번호)가 생깁니다. 앱인토스 미니앱과 별개 작업입니다.
