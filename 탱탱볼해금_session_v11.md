# 탱탱볼해금 SESSION v11 — 실기기 화면 밀림 수정 (광고 전환은 보류)

> 작성일: 2026-07-11 · 이전: `탱탱볼해금_session_v10.md` · 다음 세션은 **이 문서를 먼저** 읽을 것
> 요지: 실기기 테스트에서 발견된 "상단 바 등장 시 화면이 아래로 밀려 버튼이 안 눌리는" 문제를 수정했다. 실제 광고 전환은 **콘솔 광고 승인 대기 중**이라 보류 — 승인되면 광고 ID 교체 + 이번 수정을 묶어 새 버전으로 재심사받는다.

## 0. 이번에 한 일

| 항목 | 내용 |
|---|---|
| 화면 밀림 수정 | `app/src/hooks/useViewportSize.ts` — visualViewport 기반으로 교체, 밀림 감지 시 `scrollTo(0,0)` 자동 복구 |
| 메뉴 스크롤 허용 | `MainMenuPage.tsx`·`SettingsPage.tsx` — `overflow-y: auto` + `touch-action: pan-y` (내용이 화면보다 길어도 버튼 도달 가능) |
| 검증 | `npm run build` 통과. headless Chrome(844×390/330/300)에서 메뉴 스크롤로 하단 "스테이지 선택" 도달, 게임 캔버스 즉시 리사이즈, 콘솔 오류 0 확인 |
| 커밋 | `0a3e01a` (push 완료) |

원인 요약: 앱 전체가 `position: fixed` + 스크롤 전면 차단(`touch-action: none`) 구조라, 토스 웹뷰가 상단 바 등장 등으로 페이지를 밀면 사용자가 되돌릴 방법이 없었다. 또 구식 `resize` 이벤트만 들어서 보이는 영역 변화를 감지하지 못했다.

## 1. 남은 일 (다음 단계)

1. **광고 승인 대기** — 앱인토스 콘솔에서 광고(사업자) 승인이 나면, 실제 adGroupId 2개(전면형/보상형)를 받아 `app/src/services/ads.ts:21-24`의 테스트 ID(`ait-ad-test-*`)와 교체.
2. 광고 ID 교체 + 이번 화면 수정을 **한 버전으로 묶어** `ait build` → 콘솔 새 버전 등록 → 재심사.
3. 재심사 승인 후 실기기에서 확인할 것: ① 상단 바가 생겨도 화면이 밀리지 않는지 ② 메뉴/설정 세로 스크롤 ③ 실제 광고 노출·보상 지급.

## 2. 주의사항

- **화면 수정은 실기기(토스 앱) 미확인** — headless 브라우저 검증만 완료. 웹뷰의 밀림 시나리오는 데스크톱에서 재현 불가라 실기기 확인이 최종 검증이다.
- 광고 ID가 틀리면 광고만 안 나오고 게임은 정상(수익 0). 교체 후 샌드박스에서 노출 확인 필요.
- 코드 수정만으로는 이미 배포된 버전이 바뀌지 않는다 — 반드시 새 버전 등록·재심사 필요.
