## 2. 이번에 완료한 작업 (지금 할 수 있는 것 전부)

### 가. 앱인토스 SDK 도입 (모든 SDK 항목의 근본 해결)
- `@apps-in-toss/web-framework`(SDK 2.7.0) 설치. **기존 Vite+React 구조를 그대로 유지**하는 점진 도입(앱 통째로 다시 만들 필요 없음).
- `granite.config.ts` 추가(ait CLI 설정), `package.json`에 `ait:dev/build/deploy` 스크립트 추가.
- **핵심 안전장치:** SDK 함수는 토스 앱·샌드박스에서만 동작하고, 일반 브라우저(지금의 개발·검증)에서는 기존 방식(localStorage·UUID·가짜 광고화면)으로 자동 폴백(`src/services/sdk.ts`의 `isInTossEnv()`). → **지금 쓰던 테스트 방식이 그대로 돌아갑니다.**

### 나. 데이터·인증 (차단 #3·#4)
- **유저 식별키**(`auth.ts`): 토스 `getUserKeyForGame()`로 실제 게임 유저키 발급. 미지원 시 로컬 UUID 폴백.
- **네이티브 저장소**(`storage.ts`): 토스 `Storage`로 저장 → **앱을 껐다 켜도 진행 데이터 유지**. 브라우저에선 localStorage 폴백.

### 다. 광고 (차단/비차단 ads)
- `ads.ts`를 실제 광고로 교체. 사전 로딩(정책 준수), 보상형 시청 완료 시에만 목숨 지급, 광고 중 음악 정지·종료 후 복귀.
- 가짜 광고화면(`MockAdOverlay`)은 **브라우저에서만** 뜨도록 폴백 처리하고, 검수 반려 위험이던 "실제 출시 시 SDK로 교체됩니다" 문구를 제거했습니다.

### 라. 닫기/종료 (차단 #6·#7)
- 메뉴·설정 화면에 **✕ 닫기 버튼** 추가 → 누르면 **"게임을 종료할까요?" 확인 모달** → 종료 시 `closeView()`. 네이티브 뒤로가기도 같은 확인 모달로 연결.

### 마. 화면 제어 (권장)
- 가로 모드 고정(`setDeviceOrientation`), 플레이 중 화면 항상 켜짐(`setScreenAwakeMode`), Safe Area SDK 동기화 보강.
- 분석(`Analytics.screen`)·햅틱(`generateHapticFeedback`) SDK 교체.

### 바. 폰트 셀프호스팅 (권장)
- Pretendard·Inter를 외부 CDN 대신 `public/fonts/`에 내장(`*.woff2`) + `src/styles/fonts.css`. CDN 의존 제거로 검수 안정성↑.

### 사. 비코드 산출물 (이 폴더)
- `스토어_메타데이터_초안.md` — 부제·키워드·상세설명 초안.
- `appName_확정_제안.md` — 영문/국문 이름 후보·추천.
- `로고/logo-600.png` — 600×600 불투명 PNG(규격 충족).

## 3. 검증 결과 (근거)

- **빌드:** `npm run build` 통과(tsc 타입체크 + vite). 번들 232KB → **246KB(+14KB)** — SDK 도입에도 거의 안 늘었습니다(무거운 부분은 토스 네이티브가 처리, 웹 번들엔 안 들어옴).
- **헤드리스 실행:** `verify-browser.cjs` — 콘솔 에러 **0건**, 캔버스 렌더 정상, Stage1 실제 클리어 + 진행 저장 확인.
- **닫기/종료 모달:** `verify-exit.cjs` — 메뉴·설정 닫기 버튼 노출, 모달 표시·취소 정상, 에러 0.
- **게임 회귀:** `climb-sim`·`camera-sim` 전체 통과. (※ `validate2`/`gimmick-sim`의 S18·S20 경고는 **현재 진행 중인 게임 튜닝(S11~S20)** 사안으로, 이번 SDK/UX 작업과 무관 — 아래 5번 참조.)

## 4. ⚠️ 출시(콘솔 등록·배포) 직전 사용자 확인 필수

1. **영문 appName** — `granite.config.ts`의 `appName: 'tangtangball'`. **등록 후 수정 불가.** `appName_확정_제안.md` 참고해 최종 확정.
2. **brand.icon / primaryColor** — `granite.config.ts`. icon은 실제 로고(`로고/logo-600.png`) 경로로, primaryColor는 샌드박스에서 확인 후 조정.
3. **실제 광고 ID** — `ads.ts`의 `AD_GROUP_ID`는 현재 **테스트 ID**. 운영 광고는 **사업자 등록 필요**(가이드 §3) → 콘솔 발급 ID로 교체.

## 5. 게임이 완성되어야 할 수 있는 작업 (이번 범위에서 제외)

> 최종 게임 화면·밸런스가 확정돼야 가능한 것들입니다.

- **정식 스크린샷**(세로 636×1048 3장+, 가로 1504×741 1장+) — 최종 UI 캡처 필요.
- **썸네일**(1000×1000, 1932×828) — 최종 비주얼 합성.
- **게임 등급분류 신청**(1~2주) — 최종 플레이 화면 + 사업자명 일치 필요. (방법 결정·사업자 준비는 지금 가능)
- **실기기 QA / 메모리 프로파일 / 저사양 FPS** — 최종 빌드로 실측.
- **토스 샌드박스 검증** — `ait dev`로 닫기·유저키·가로고정·광고·Safe Area 실제 동작 확인. (특히 광고 SDK 호출과 네이티브 닫기 버튼 연동은 샌드박스 실측 권장)
- **진행 중인 게임 튜닝(S11~S20 난이도, S18/S20 도달성)** — 사용자 작업 영역.

## 6. 실행·배포 방법

```bash
cd app
npm run dev          # 기존 브라우저 테스트 (그대로 동작)
npm run build        # 프로덕션 웹 번들

npm run ait:dev      # 토스 샌드박스 연결 개발 (SDK 실동작 확인)
npm run ait:build    # 앱인토스 번들 빌드
npm run ait:deploy   # 콘솔 업로드용 배포 (이름/아이콘/광고 ID 확정 후)
```
