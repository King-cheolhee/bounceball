# 게임용 상단 내비게이션 전환 설계

작성일: 2026-07-11
대상: 앱인토스 WebView 게임 `탱탱볼해금`

## 1. 목적

현재 iPhone과 Android 가로 화면에서 비게임용 흰색 내비게이션 바가 세로 공간을 차지한다. 앱 유형에 맞는 게임용 내비게이션으로 전환해 흰색 바, 뒤로가기, 로고, 앱 이름을 없애고 게임 화면의 사용 가능 높이를 확보한다.

성공 기준은 다음과 같다.

- iPhone과 Android 모두 게임 화면이 흰색 상단바 없이 표시된다.
- 토스가 제공하는 게임용 `더보기`와 `닫기(X)`는 우측 상단에 유지된다.
- 게임 내부 버튼이 토스의 `더보기`와 `닫기(X)`에 겹치지 않는다.
- 기존 진행 저장, 광고, 게임 물리, Safe Area 동기화는 바뀌지 않는다.

## 2. 검토한 방법

### A. 게임용 내비게이션으로 전환하고 중복 닫기 버튼 정리, 채택

`granite.config.ts`에 `webViewProps.type: 'game'`을 설정한다. 토스 공식 게임용 내비게이션이 적용되며, 화면 내부에 중복으로 존재하는 `✕ 닫기` 버튼은 메뉴와 설정 화면에서 제거한다.

- 장점: 게임 유형과 심사 가이드에 맞고, iPhone과 Android가 같은 프레임워크 경로를 사용한다.
- 단점: 네이티브 환경에서만 최종 모양을 확인할 수 있어 샌드박스 실기기 검증이 필요하다.

### B. 비게임용 내비게이션을 투명하게 변경, 제외

흰색 배경과 제목을 숨기는 커스터마이징을 적용하는 방법이다.

- 제외 이유: 현재 설치된 WebView SDK 2.7.0의 설정 표면과 맞지 않는 항목이 있고, 등록된 서비스가 게임이므로 비게임 유형을 유지할 이유가 없다.

### C. 현재 상단바를 유지하고 게임 UI를 더 축소, 제외

내부 여백과 글자 크기를 더 줄여 흰색 바 아래에 내용을 맞춘다.

- 제외 이유: 작은 iPhone 가로 화면에서 가독성을 계속 희생하고, 플랫폼 유형 오류를 그대로 남긴다.

## 3. 변경 설계

### 앱인토스 설정

`app/granite.config.ts`에 다음 항목을 추가한다.

```ts
webViewProps: {
  type: 'game',
},
```

현재 프레임워크 기본값은 `partner`이므로 설정이 없으면 비게임용 흰색 바가 표시된다. `game`으로 명시하면 게임용 내비게이션으로 패키징된다.

### 중복 닫기 UI

다음 화면의 자체 `✕ 닫기` 버튼만 제거한다.

- `MainMenuPage`: 설정 버튼과 자원 정보는 유지한다.
- `SettingsPage`: 메뉴로 돌아가기 버튼은 유지한다.

토스의 게임용 `X`가 모든 화면에서 미니앱 종료 경로를 제공한다. 기존 네이티브 뒤로가기 이벤트의 종료 확인 모달과 `closeMiniApp()` 서비스는 삭제하지 않는다. 따라서 OS 뒤로가기 처리와 브라우저 폴백 구조는 유지된다.

### Safe Area

기존 `SafeAreaInsets.get/subscribe`와 `--safe-top/right/bottom/left` 적용을 유지한다. 새 화면 전용 좌표 계산이나 기기별 분기는 추가하지 않는다. 토스의 게임용 버튼은 프레임워크가 배치하고, 앱 내부 우측 상단 닫기 버튼을 제거해 충돌 가능성을 없앤다.

## 4. 영향 범위

변경되는 영역:

- 앱인토스 패키징 설정
- 메뉴와 설정 화면의 중복 닫기 버튼 및 관련 전달 속성

변경되지 않는 영역:

- 게임 물리, 스테이지, 렌더링, 입력
- 저장 데이터와 해금 데이터
- 광고 ID와 광고 노출 흐름
- 가로 모드 고정과 Safe Area SDK 동기화
- 네이티브 뒤로가기 종료 확인 모달

잘못되면 생길 수 있는 문제는 게임용 내비게이션이 적용되지 않거나, 우측 상단 네이티브 버튼이 내부 UI와 겹치는 것이다. 데이터 손실이나 게임 밸런스 변화 가능성은 없다.

## 5. 검증

로컬 검증:

1. TypeScript 타입 검사.
2. Vite 프로덕션 빌드.
3. 데스크톱 가로 뷰포트에서 메뉴, 설정, 플레이 진입과 내부 버튼 확인.
4. `.ait` 번들 생성 후 패키지 설정에 게임 유형이 포함됐는지 확인.

실기기 검증:

1. iPhone 가로 모드에서 흰색 상단바 제거, 다이내믹 아일랜드와 우측 버튼 겹침 없음.
2. Android 가로 모드에서 흰색 상단바 제거, 뒤로가기 종료 확인, 메뉴와 설정 조작 가능.
3. 두 플랫폼 모두 토스 `더보기`와 `X`가 정상 동작.
4. 게임 시작, 좌우 입력, 스테이지 복귀, 광고 진입에 회귀 없음.

## 6. 출시 절차

최신 공개 버전은 그대로 보존한다. 변경 후 새 `.ait` 번들을 만들고 앱인토스 콘솔에 새 버전으로 등록해 검토를 요청한다. 검토 승인 전에는 기존 공개 버전이 계속 서비스되며, 승인 후 최종 `출시하기` 단계로 새 버전을 공개한다.

## 7. 근거

- 앱인토스 공식 설정 문서: 게임 WebView는 `webViewProps.type: 'game'` 사용.
- 앱인토스 공식 내비게이션 문서: 게임용 내비게이션은 우측 `더보기`와 `닫기(X)`로 구성.
- 앱인토스 공식 Safe Area 문서: 게임 내 버튼이 프레임워크 X 버튼과 겹치면 검수 반려 가능.
