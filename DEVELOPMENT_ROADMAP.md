- [ ] 앱인토스 콘솔 가입 ([apps-in-toss.toss.im](https://apps-in-toss.toss.im/))
- [ ] 워크스페이스 생성
- [ ] 토스 비즈니스 인증 (사업자 있을 시)

- [ ] 로컬 서버 ↔ 샌드박스 앱 연결 테스트

- [ ] Sentry 프로젝트 생성 (선택, 추후 연동)

#### 0.4 폴더 구조 생성
- [ ] `TECHNICAL_SPEC.md`의 구조에 맞춰 빈 폴더 생성
- [ ] CSS 변수 (`variables.css`) 정의:
  ```css
  :root {

    --color-fg: #FFFFFF;

    --font-text: 'Pretendard', system-ui, sans-serif;
  ```

### 완료 기준
✅ `npm run dev` → 빈 React 화면이 샌드박스 앱에서 정상 로드됨

## Phase 1: 게임 코어 (Day 2~6)

### 작업 목표
게임의 핵심 메커니즘 (공의 물리, 충돌, 카메라) 완성

### 체크리스트

#### 1.1 Canvas 기반 게임 루프
- [ ] `GameEngine.ts` 작성: `requestAnimationFrame` 루프
- [ ] `Renderer.ts`: Canvas 2D 컨텍스트 초기화 및 그리기 헬퍼
- [ ] 가로 모드 강제 + Safe Area 적용
- [ ] 60 FPS 목표 측정 도구 추가 (개발 모드 한정)

#### 1.2 공 (Ball) 구현
- [ ] 기본 원형 렌더링
- [ ] 중력 적용 → 자동 낙하
- [ ] 바닥 충돌 시 자동 반사 (위로 튕김)
- [ ] **관성 좌우 이동** 구현 (가속/감속/최대속도)
- [ ] 벽 충돌 → 반사
- [ ] 짧은 모션 트레일 (선택)

#### 1.3 입력 처리
- [ ] `useTouchInput` 훅 작성
- [ ] 화면 좌/우 반반 분할 터치 인식
- [ ] 멀티 터치 지원 (양손 동시 입력)
- [ ] 입력 떨어졌을 때 관성 모드 진입

#### 1.4 바닥 & 함정 엔티티
- [ ] `Floor.ts`: 일반/부서지는/폭발 발판 3종
- [ ] `Spike.ts`: 위에서 닿으면 사망
- [ ] `Wall.ts`: 측면 벽 (반사)
- [ ] 충돌 감지 로직 (`Collision.ts`)

#### 1.5 카메라 (축 스크롤)
- [ ] 공이 화면 일정 지점 지나면 카메라가 따라감
- [ ] 부드러운 스크롤 (lerp 적용)
- [ ] 스테이지 끝에서 카메라 정지

#### 1.6 사망/클리어 판정
- [ ] 공이 구멍에 빠짐 → 사망
- [ ] 공이 가시에 닿음 → 사망
- [ ] 폭발 발판 밟음 → 사망
- [ ] 공이 스테이지 끝 도달 → 클리어

### 완료 기준
✅ 단일 테스트 스테이지에서 공이 자연스럽게 움직이고, 함정에 빠지면 사망 처리됨

## Phase 3: 스테이지 설계 (Day 10~12)

### 작업 목표
20개 스테이지 데이터 작성 + 속도 곡선 + 체크포인트 시스템 구현
> 상세 설계는 `STAGE_DESIGN.md` 참조

### 체크리스트

#### 3.1 스테이지 데이터 포맷 확정
- [ ] `stages.json` 스키마 정의 (`StageData` 인터페이스 기반)
- [ ] `StageLoader.ts` 로딩 로직 구현
- [ ] 스테이지 검증기 작성 (잘못된 데이터 방지)

#### 3.2 속도 곡선 시스템
- [ ] `getBouncePeriod(stage)` 함수 구현 (Stage 1: 1.00초 → Stage 20: 0.40초)
- [ ] `getPhysicsForStage(stage)` 함수 구현 (중력/바운스속도/최대속도 동적 계산)
- [ ] Ball 클래스의 `setStage(stage)` 메서드 구현
- [ ] 스테이지 진입 시 물리 상수 갱신 호출

#### 3.3 체크포인트 시스템 ⭐
- [ ] `checkpoint_stage` Storage 키 추가
- [ ] `onStageClear(clearedStage)` 함수 작성 (5/10/15 클리어 시 체크포인트 자동 저장)
- [ ] `onGameOverReturnToCheckpoint()` 함수 작성
- [ ] 메인 메뉴 진입 시 `checkpointStage`로부터 시작 로직
- [ ] 체크포인트 도달 시 시각적 피드백 (선택, 예: "CHECKPOINT" 텍스트 1초 표시)

#### 3.4 스테이지 디자인 (`STAGE_DESIGN.md` 5장 기반)
- [ ] **튜토리얼 (1~5)**: 조작법 학습
  - Stage 1: 첫 발걸음 (평지)
  - Stage 2: 첫 구멍
  - Stage 3: 두 번째 구멍 (연속 점프)
  - Stage 4: 첫 가시
  - Stage 5: 조합 첫 단계 (구멍 + 가시) — **체크포인트 1 트리거**
- [ ] **기본 (6~10)**: 부서지는 바닥 도입
  - Stage 6: 부서지는 바닥 등장 — 체크포인트 1 시작점
  - Stage 7: 부서지는 바닥 연속
  - Stage 8: 공중 부서지는 발판
  - Stage 9: 정밀 조작 입문
  - Stage 10: 챕터 2 보스 — **체크포인트 2 트리거 + 전면 광고 1회**
- [ ] **응용 (11~15)**: 폭발 발판 + 천장 가시
  - Stage 11: 폭발 발판 등장 — 체크포인트 2 시작점
  - Stage 12: 폭발 + 부서짐
  - Stage 13: 좁은 안전 구간
  - Stage 14: 천장 가시 도입
  - Stage 15: 챕터 3 보스 — **체크포인트 3 트리거 + 전면 광고 1회**
- [ ] **마스터 (16~20)**: 극한 속도 + 복합 함정
  - Stage 16: 마스터 입문 — 체크포인트 3 시작점
  - Stage 17: 교차 패턴
  - Stage 18: 지옥의 회랑
  - Stage 19: 최종 시험
  - Stage 20: 그랜드 마스터 — **게임 완전 클리어**

#### 3.5 개발자 도구 (디버깅용)
- [ ] 키보드 단축키로 스테이지 점프 (개발 모드 한정)
- [ ] 좌표 표시 / FPS 표시
- [ ] 무적 모드 토글 (스테이지 튜닝용)
- [ ] `import.meta.env.DEV` 가드 확실히 적용 (운영 빌드 제외)

#### 3.6 자체 플레이 테스트
- [ ] 20개 스테이지 모두 한 번씩 클리어
- [ ] 챕터별 클리어율 자가 측정 (목표: 튜토 90%, 기본 70%, 응용 50%, 마스터 30%)
- [ ] 체크포인트가 의도대로 작동하는지 확인
- [ ] 속도 곡선 체감 확인 (스테이지 10, 15, 20에서 "빨라졌다" 느낌)

### 완료 기준
✅ 20개 스테이지 모두 클리어 가능 + 체크포인트 자동 저장/복원 정상 작동 + 속도 곡선 체감

## Phase 6: 마감 & 테스트 (Day 18~20)

### 작업 목표
검수 통과를 위한 최종 점검 및 폴리싱

### 체크리스트

#### 6.1 게임 출시 체크리스트 점검

##### 접속 및 사운드
- [ ] 10초 이내 최초 화면 노출
- [ ] 사운드 ON/OFF 사용자 설정 가능
- [ ] 무음/진동꺼짐 모드에서도 정상 작동
- [ ] 백그라운드 전환 시 사운드 즉시 중지
- [ ] 백그라운드 복귀 시 사운드 정상 재생

##### UX 및 내비게이션 바
- [ ] 우측 상단 닫기 버튼 정상 동작
- [ ] Safe Area 침범 없음 (Dynamic Island 포함)
- [ ] 진입 즉시 바텀시트 노출 없음
- [ ] 모든 화면에서 미니앱 종료 경로 존재
- [ ] CTA 버튼 의도 명확

##### 보안 및 안정성
- [ ] `eval` 등 외부 코드 실행 함수 미사용
- [ ] `window.location.replace` 등 히스토리 조작 미사용
- [ ] CSR/SSG만 사용 (SSR 없음)

##### 서비스 이용 동작
- [ ] 인게임 풀스크린 구현
- [ ] 가로 모드 정상 작동
- [ ] OS 뒤로가기 제스처 비활성화
- [ ] 모든 UI 컴포넌트 정상 작동
- [ ] 인터랙션 반응 2초 이내
- [ ] 미니앱 종료 시 확인 모달

#### 6.2 성능 점검
- [ ] 60 FPS 유지 (저사양 기기 포함)
- [ ] 메모리 사용량 비정상 급증 없음
- [ ] 번들 크기 < 100MB
- [ ] 첫 진입 ~ 게임 시작까지 5초 이내

#### 6.3 디바이스 테스트
- [ ] iPhone (iOS 16 이상)
- [ ] Android (Android 7 이상)
- [ ] 다양한 화면 비율 (16:9, 19.5:9, 노치 포함)
- [ ] Safe Area 모든 기기에서 정상
