## 1. 서비스 오픈 정책 (출시 제한 사항)

### 출시 불가 서비스
다음 서비스는 토스 플랫폼 정책상 출시할 수 없습니다.

- **디지털/가상자산 관련**: NFT 포함, 디지털 자산 소유·이전·저장·거래·발행
- **자금세탁 가능성**: 현금/유사자산의 직접 교환·전환·환불 기능
- **불법/부정행위 조장**: 신분 조작, 해킹, 불법 문서 제공 등
- **사행성 및 복권/베팅성 콘텐츠**
- **금융 상품 중개·판매·광고**
- **투자 자문, 리딩방, 유료 정보 제공**
- **의료 관련 서비스**: 비대면 진료, 병원 예약·홍보 등
- **단순 홍보 목적**의 기존 회사/서비스 미니앱

### 어뷰징 방지 정책
- **동일 워크스페이스 내**에서 유사한 핵심 기능 기반 앱을 여러 개 출시 불가
- 테마/대상/주제만 다른 미니앱(예: 'AI 여자 얼굴 만들기', 'AI 남자 얼굴 만들기')은 별도 미니앱으로 인정되지 않음
- 동일 브랜드로 여러 미니앱 출시 불가 (단, 서비스 목적이 명확히 다르면 가능)

### 자사 앱 설치/외부 링크 제한
- **자사 앱 설치 유도 금지**: "앱 설치하면 더 많은 기능", "전용 쿠폰 받으려면 앱 설치" 등 모두 금지
- **외부 링크 이동 제한**: 미니앱 기능은 미니앱 내에서 완결되어야 함
- 허용 예외: 법률상 고지, 공공기관/제휴기관 페이지, 최저가 구매 플랫폼 이동 등

### 로그인/결제/광고 연동 정책
- **로그인**: 토스 로그인만 사용 가능 (소셜/간편 로그인 사용 불가)
- **결제**:
  - 실물 상품 → 토스페이만
  - 디지털 상품 → 인앱결제만
- **광고**: 앱인토스 전면형/보상형/배너 광고만 (외부 광고 네트워크 금지)

### 생성형 AI 사용 시
- **사전 고지 의무**: 최초 진입 시 AI 활용 사실 고지
- **표시 의무**: 결과물에 AI 생성임을 라벨/배지/워터마크로 표시
- 위반 시 최대 3,000만 원 과태료 가능

## 2. 전체 출시 프로세스

### 개발 기간
- **게임**: 약 2~4주
- **비게임**: 약 1~3개월

### 단계별 흐름

1. **시작하기**: 콘솔 회원가입 → 워크스페이스 설정 → 멤버 초대 → 앱 등록
2. **디자인하기**: 앱빌더 또는 피그마(Figma) 활용 + TDS 사용
3. **개발하기**: SDK + API 활용 (Granite 기반)
4. **검수하기**: 4단계 검수 (운영 → 기능 → 디자인 → 보안), 평균 2~3일
5. **출시**: 콘솔에서 '출시하기' 버튼

### ⚠️ 중요: SDK 2.x 마이그레이션
- **2026년 3월 23일 이후 SDK 1.x로 빌드한 번들은 콘솔 업로드 불가**
- RN 0.84 테스트는 2026년 3월 6일 업로드된 최신 샌드박스 앱에서 진행

### 사용자/기기 환경
- **사용자**: 만 19세 이상
- **기기**: Android 7 이상 또는 iOS 16 이상

## 3. 콘솔 앱 등록 (게임 등급분류 포함)

### 가입 조건
- 만 19세 이상
- 본인 명의로 로그인된 토스앱 필요
- 토스 비즈니스 회원 기반

### 워크스페이스 / 권한
- 사업자당 1개 워크스페이스
- **대표관리자**: 워크스페이스 운영 책임, 약관 동의 권한
- **관리자**: 멤버 관리, 설정 변경 등
- **구성원**: 공동 작업만 가능

### 사업자 없이 출시 가능
- 사업자 등록 없이도 앱 출시 가능
- **단, 다음 기능은 사용 불가**: 토스 로그인, 비즈월렛, 프로모션, 인앱 광고, 토스페이, 인앱 결제

### 앱 기본 정보
- **앱 로고**: 600 x 600px, PNG (투명 배경 불가)
- **앱 이름**: 한글 + 영문(15자 이내 명사형, 명령문/동사구 금지)
- **appName**: 한 번 등록 후 수정 불가 (앱 스킴 `intoss://` 형식)
- **사용 연령**: 만 19세 이상
- **고객센터 정보**: 이메일/연락처/채팅 상담 주소

### 게임 등급분류 (필수)

게임은 출시 전 반드시 등급 심의를 완료해야 합니다.

**방법 1: 오픈마켓 자체등급분류 활용**
- 앱스토어, 구글 플레이, 원스토어, Microsoft Store에서 등급 분류받은 경우
- 입력 필요 정보:
  - 스토어 출시 게임 페이지 URL
  - 등록자명 (구글: 개발자명 / 애플: 제공자명)
  - 자체등급분류사업자명, 등급분류일자, 등급분류번호
  - 이용등급, 내용정보
  - 대표자 인감/사인 이미지
  - 게임 플레이 화면 (자체등급분류 게임 2장 + 앱인토스 게임 2장)

**방법 2: 게임물관리위원회 직접 심의**
- 게임물 등급분류증명서 PDF 첨부

> 🚨 게임물 등록자명과 사업자명이 다르면 검수에서 반려됨

### 카테고리 및 노출 정보
- **썸네일**: 정방형 1000x1000px / 가로형 1932x828px (PNG)
- **부제**: 비속어/느낌표 사용 불가
- **상세 설명**: 토스 홈 광고 대상 선정 기준이 됨 → 구체적으로 작성
- **앱 검색 키워드**
- **스크린샷**: 세로형 636x1048px(최소 3장) / 가로형 1504x741px(최소 1장)
- **리더보드**: 점수 단위 및 정렬 기준 설정

### 검토 소요
- 앱 정보 검토: 영업일 기준 **1~2일**
- 사업자 등록 검수: 영업일 기준 **1일**

## 4. 게임 출시 검수 체크리스트

### 접속 및 사운드
- [ ] 미니앱이 10초 이내에 최초 화면까지 정상적으로 열림
- [ ] 배경음, 효과음, 햅틱 등이 적용되어 있음 (무음 모드/진동 꺼짐 상태에서도 의도한 동작 유지)
- [ ] 사운드가 있는 경우 사용자가 On/Off 직접 설정 가능
- [ ] 백그라운드 전환 시 사운드 즉시 종료
- [ ] 백그라운드 → 복귀 시 사운드 정상 재생

### UX 및 내비게이션 바
- [ ] 우측 상단 닫기 버튼 정상 동작
- [ ] Safe Area 영역 침범 안 함 (iOS Dynamic Island 포함)
- [ ] 진입 즉시 바텀시트 자동 노출 안 함
- [ ] 화면 전환 시 바텀시트로 사용자 행동 강제 유도 안 함
- [ ] 모든 화면에서 미니앱을 나갈 수 있는 방법이 있음
- [ ] CTA 버튼만 봐도 다음 행동을 예측 가능
- [ ] 자사 서비스/앱 설치 유도 없음
- [ ] 인앱 광고는 인트로/로딩/컷신/팝업 모달 등 일시적 화면에 노출 안 함
- [ ] 브랜드 로고와 미니앱 이름(국문)을 내비게이션 바에 반영

### 유저 식별키 발급
- [ ] 유저 식별자 값 확인·저장 후 게임 정상 시작
- [ ] 유저 플레이 기록(랭킹, 레벨, 스테이지) 유지
- [ ] 미니앱 종료 후 재접속해도 데이터 유지

### 보안 및 안정성
- [ ] 외부 코드 실행 기능 사용 안 함 (예: `eval`)
- [ ] 브라우저 히스토리 조작 안 함 (예: `window.location.replace`로 자사 사이트 이동)
- [ ] **SSR 불가** → CSR 또는 SSG 방식만 사용

### 서비스 이용 동작
- [ ] 인게임 화면 풀스크린 구현
- [ ] 가로/세로 모드 의도대로 작동
- [ ] **OS 뒤로가기 제스처 사용 불가**
- [ ] 불법성/선정성 콘텐츠 없음
- [ ] 모든 UI 컴포넌트 정상 작동
- [ ] 스크롤/터치/화면 전환 2초 이상 지연 없음
- [ ] (RN) 권한 요청 전 사용자 동의 먼저 받음
- [ ] 권한 거부해도 나머지 기능 정상 작동
- [ ] 네트워크/메모리 사용량 비정상 급증 없음
- [ ] 미니앱 종료 시 확인 모달 노출
- [ ] 점수 측정, 스테이지 클리어 등 주요 흐름 오류 없음

### 인앱 결제
- [ ] 결제 시 미니앱 음악 일시정지
- [ ] 주문 금액과 구글/애플 결제창 금액 일치
- [ ] 구글 결제 테스트 환경에서도 정상 진행
- [ ] 결제 완료 후 결제 결과 정상 반영
- [ ] 결제 취소 시 주문 화면으로 복귀
- [ ] 결제 실패 시 사용자가 실패 사유 인지 가능
- [ ] 결제 취소 정상 처리
- [ ] 결제 내역 확인 가능
- [ ] 기기 변경해도 결제 데이터(아이템) 유지

### 인앱 광고
- [ ] 광고 재생 중 미니앱 음악 일시정지
- [ ] 예상하기 어려운 순간에 광고 노출 안 함
- [ ] 광고 정상 노출, 중간 종료/튕김 없음
- [ ] **광고 사전 로딩** (재생 시점 실시간 로딩 금지)
- [ ] 광고 종료 후 미니앱 화면으로 복귀
- [ ] 광고 종료 후 음악 다시 재생
- [ ] 리워드 광고 완전 시청 시 보상 정상 지급
- [ ] **배너 광고는 상단/하단에만 위치**

### 공유 리워드
- [ ] 공유 리워드 화면 정상 노출
- [ ] 닫으면 미니앱으로 정상 복귀
- [ ] 친구 초대 완료 시 보상 정상 지급

### 검수 소요
- 앱 출시 검수: **영업일 기준 3일**
- 앱 번들 용량 제한: 압축 해제 기준 **100MB 이하**

## 5. 다크패턴 방지 정책

다음 사례는 출시 불가입니다.

1. **서비스 진입 즉시 바텀시트 노출** (광고성, 알림 동의 등 포함)
2. **뒤로 가기 시 알림 동의 바텀시트로 이탈 차단**
3. **나갈 수 있는 선택지가 없는 강제 구조**
4. **예상치 못한 순간의 전면 광고 노출**
5. **CTA 버튼만으로 다음 행동 예측 불가** (라벨이 모호하거나 설명만 반복)

## 6. SDK 핵심 기능 (게임용)

### SDK 구조
- 코어: **Granite** (앱 실행 환경 초기화, 토스 앱과 통신)
- 두 가지 형태 제공: **WebView SDK** / **React Native SDK**
- `npx create-ait-app <app-name>`으로 시작

### 게임 미니앱 필수 기능

| 카테고리 | 기능 | 권장도 | 설명 |
|----|----|----|----|
| 인증 | 게임 로그인 (`getUserKeyForGame`) | 필수 | 게임 전용 유저 식별자 발급 |
| 게임 | 게임 리더보드 (`submitGameCenterLeaderBoardScore`) | 필수 | 점수 제출·순위 확인 |
| 디바이스 | 네이티브 저장소 (`Storage`) | 필수 | 유저 정보 저장 (기기 변경 시 유실 방지 설계) |
| 화면 제어 | Safe Area 여백 구하기 | 필수 | 디바이스별 UI 가림 방지 |
| 분석 | 사용자 행동 기록 | 필수 | 이벤트 추적, A/B 테스트 |

### 게임 권장 기능

| 기능 | 설명 |
|----|----|
| 프로모션 (`grantPromotionRewardForGame`) | 토스 포인트 보상 지급 |
| 공유 리워드 (`contactsViral`) | 친구 초대 바이럴 |
| 토스앱 공유 링크 (`getTossShareLink`) | 결과/초대 링크 생성 |
| 인앱 결제 (IAP) | 아이템 판매 |
| 전면형/리워드/배너 광고 | 광고 수익화 |
| 화면 항상 켜짐 (`setScreenAwakeMode`) | 방치형/지속형 게임 |
| 화면 방향 설정 (`setDeviceOrientation`) | 가로/세로 고정 |
| 오디오 포커스 콜백 (`OnAudioFocusChanged`) | 사운드 자연스러운 처리 |
| 햅틱 진동 (`generateHapticFeedback`) | 몰입도 향상 |
| 서버 시간 (`getServerTime`) | 클라이언트 시간 조작 방지 |

### 개발 시 주의사항
- **iframe 사용 불가** (단, YouTube 영상은 예외)
- **SSR 불가** → CSR/SSG만
- **mTLS 인증 필수**: 서버 간 통신 시 (미적용 시 `ERR_NETWORK`)
- API 서버 CORS 허용 목록에 앱인토스 도메인 추가 필요
- **샌드박스 테스트 필수** (브라우저 테스트로는 부족)
- 다크 모드 미지원 → 라이트 모드 기준 개발
- Storage 데이터는 앱 삭제 시 함께 삭제됨 (영속 필요 시 자체 서버)

---

## 7. 유저 식별 (게임 로그인)

### 게임 로그인 vs 토스 로그인

| 구분 | 토스 로그인 | 유저 식별키 발급 |
|----|----|----|
| 사용 가능 대상 | 모든 제휴사 | 모든 미니앱 |
| 사용 함수 | `appLogin` | `getUserKeyForGame`(게임), `getAnonymousKey`(비게임) |
| 서버 구축 | 필요 | **불필요** |
| 콘솔 설정 | 필요 | **불필요** |
| 유저 동의 | 필요 | **불필요** |

### 게임에서 사용
- 게임 미니앱은 **`getUserKeyForGame`** 사용
- 별도 서버 없이 사용자 식별 가능
- 동의 절차 없이 즉시 서비스 시작 가능

---

## 8. 게임 센터 (프로필 & 리더보드)

### 게임 프로필
- 게임 시작 전 사용자가 프로필을 한 번 등록
- 최초 1회만 설정하면 이후 진입 시 환영 메시지 자동 노출
- **프로필 미생성 사용자는 게임 진행 불가**
- 프로필에는 사용자 식별자 미포함 (필요 시 토스 로그인 별도 연동)

### 리더보드
- 사용자 점수 기반 랭킹 기능
- 친구 맺기, 점수 공유 가능
- **점수 제출은 게임 시작이 아닌 게임 종료 후 호출**

### 효과
- 자연스러운 온보딩 흐름
- 랭킹/친구 경쟁으로 리텐션 강화
- 점수 공유로 바이럴 효과

## 9. 수익화: 인앱 결제

### 상품 유형
- **소모성**: 사용하면 사라짐 (게임 아이템, 코인, 힌트 이용권)
- **비소모성**: 한 번 구매로 영구 사용 (광고 제거, 프리미엄 기능)

### 가격 제한
- **최소 400원 ~ 최대 1,400,000원**

### 환불 처리
- **Google Play**: 48시간 이내 자동, 이후 수동 요청
- **App Store**: 모든 환불은 Apple이 직접 처리

### 참고
- 결제 시 음악/영상은 일시정지, 결제 후 자동 복귀 처리 필수
- 판매가 = 공급가 + 부가가치세(VAT)

## 10. 수익화: 인앱 광고

### 광고 유형
- **전면형 광고**: 화면 전체 덮음, 화면 전환 지점에 노출
- **보상형 광고**: 시청 시 보상 지급, 사용자 선택형
- **배너 광고**: 상단/하단/중앙 위치 (단, 게임은 상/하단만 허용)

### 테스트용 ID
운영 ID 사용 시 제재 가능. 반드시 테스트용 ID 사용:
- 전면형: `ait-ad-test-interstitial-id`
- 보상형: `ait-ad-test-rewarded-id`
- 배너 리스트형: `ait-ad-test-banner-id`
- 배너 피드형: `ait-ad-test-native-image-id`

### 주의사항
- 광고 재생 중 앱 사운드 일시정지 → 종료 후 자동 재생
- 과도한 광고 노출 금지
- **광고 사전 로딩 필수** (재생 시점 실시간 로딩 금지)

## 11. 정산 구조

### 정산 등록 정보
- 사업자 정보
- 거래 계좌 사본
- 세금계산서 발행용 이메일
- **사업자 단위로 합산 정산** (앱 단위 아님)

### 인앱 광고 정산

**광고 매출 흐름 (외부 미디에이션의 경우)**
```
광고 매출 ₩10,000 
  → 외부 미디에이션 30% 공제 
  → 광고 수익 ₩7,000 (+ VAT ₩700)
  → 앱인토스 수수료 15% 공제 (CBT 0%: 2026년 3월 31일까지)
  → 파트너사 정산금
```

**예시 (앱인토스 수수료 적용 시)**

| 광고 수익 | VAT | 앱인토스 수수료 | 수수료 VAT | 파트너사 정산금 |
|----|----|----|----|----|
| ₩14,700 | ₩1,470 | ₩2,205 | ₩221 | **₩13,774** |

**정산 일정**
- 정산 내역 확정: 익월 2영업일 오후 12시
- 정산금 지급: 익월 말 영업일

**세금계산서 (역발행)**
- 광고 수익 부가세: 토스가 파트너사로 역발행 → 파트너사가 팝빌에서 승인
- 승인 마감: 매월 4영업일까지 (미승인 시 정산 지급 이월, 2회 이상 미승인 시 미니앱 운영 중단 가능)
- 사전 준비: 사업자 공동인증서 + 팝빌 회원가입

### 인앱 결제 정산

**수수료 구성**
- **앱마켓 수수료**: 공급가 기준 15% (현재) 또는 30%
- **토스 수수료**: 결제 금액 기준 5% (CBT 기간 0%)

**예시: ₩11,000 결제 (VAT 포함)**

| 마켓 | 마켓 수수료 | 토스 수수료 | 토스 VAT | 파트너사 정산금 |
|----|----|----|----|----|
| 애플 | ₩1,650 (VAT 포함) | ₩550 | ₩55 | **₩8,745** |
| 구글 | ₩1,500 | ₩550 | ₩55 | **₩8,895** |

**정산 일정**
- 정산 내역 제공: 익월 5일 이내
- 정산금 지급: 익월 말 영업일

**세금계산서**
- 애플 수수료 VAT: 토스가 대신 발행 (애플 정산금에서 선공제, 추후 환급)
- 토스 수수료 VAT: 토스가 선공제 후 직접 발행
- 구글 인앱 결제는 별도 부가세 없음

### 현금영수증 (구글 기프트카드)
- 결제일 기준 5일 이내 발행 의무
- **2025.12.01 이후 토스가 자동 대행 발행**
- 앱마켓 직권취소 시에도 현금영수증 취소하지 말 것

### 비즈월렛
- 프로모션/광고 푸시 운영용 선충전 금액
- 신용카드(법인카드 포함)로 충전
- 인보이스 익월 1~2일 사이 업데이트

### 해외 파트너사
- 매출은 KRW로 집계
- 정산금은 USD 또는 KRW(자유원 계좌 등록 시)
- 한국 발생 송금 수수료는 토스 부담, 해외 구간은 파트너사 부담

## 12. Unity 게임 적용 가이드

Unity로 만든 게임은 WebGL 빌드를 통해 앱인토스 미니앱으로 전환 가능합니다.

### 호환성 ✅ (완전 지원)
- **렌더링**: Built-in/URP/HDRP, WebGL 1.0/2.0, Standard/Lit/Unlit 셰이더, 파티클(VFX Graph 포함)
- **스크립팅**: C# (.NET Standard 2.1), IL2CPP, 코루틴, JSON 직렬화
- **물리**: 2D/3D 물리, 충돌 감지, Joint, Raycast
- **UI**: uGUI, UI Toolkit, Input System, TextMeshPro

### 호환성 ⚠️ (제한적)
- **네트워킹**: HTTP/HTTPS, WebSocket(써드파티 필요)
- **파일**: Resources, StreamingAssets(CDN), PlayerPrefs(LocalStorage 매핑)
- **멀티미디어**: 오디오(WebM/OGG 권장), 비디오(H.264/WebM)
- **센서**: GPS(정확도 낮음), 진동(SDK 지원)

### 호환성 ❌ (미지원)
- TCP/UDP Socket, P2P 통신
- 로컬 파일 I/O
- ARCore/ARKit
- Native Plugin, DLL Import, 플랫폼 분기 코드(#if UNITY_IOS 등)
- Compute Shader(제한적)

### 성능 평가 기준
- **메모리**: < 128MB(좋음), 128~256MB(보통), > 256MB(주의)
- **Draw Call**: < 100(적합), 100~300(최적화 필요), > 300(대대적 최적화)

### 게임 장르별 적합성

**매우 적합** 🟢
- 퍼즐, 캐주얼, 카드/보드 게임

**적합** 🟡 (최적화 필요)
- 액션, RPG, 시뮬레이션

**도전적** 🔴
- FPS/TPS, 레이싱, MMO

### 전환 권장 조건
- Unity 2021.3 LTS 이상
- 메모리 사용량 < 200MB
- 네이티브 플러그인 미사용
- 웹 친화적 디자인

### Unity SDK 주요 API
- 인증, 시스템 정보, 공유, 스토리지, 내비게이션, Safe Area, 게임 센터
- 권한: 클립보드, 위치, 미디어, 연락처
- 빌드 프로필 (개발/테스트/배포)
- 인앱 광고, 인앱 결제 연동
- Sentry 통합 지원

### 최적화 영역
- 시작 최적화: 게임 시작 속도, 시작 시간, 첫 씬 최적화
- 로딩 최적화: 코드 분할(wasm-split), 커스텀 로딩 화면
- 리소스 로딩: AssetBundle, Addressable, AutoStreaming, 사전 다운로드
- 렌더링: URP 커스터마이징, WebGL 2.0, EmscriptenGLX

## 13. 웹보드 게임 (해당 시)

> 포커, 고스톱, 맞고, 체스, 장기 등 보드/카드/전략형 게임

콘솔에서 **앱 정보 등록 시 반드시 웹보드 여부 체크**

### 법적 요구사항
- **게임물관리위원회(GRAC) 등급분류 필수** (앱스토어/구글플레이 IARC 등급은 국내 효력 없음)
- 사행성 방지: 베팅·도박·재산상 이익 유도 금지
- 청소년 보호: 청소년유해매체물 표시, 본인 인증, 이용 제한
- 개인정보 보호 안전성 확보

### 운영 요구사항
- 고객센터 운영 및 분쟁 해결 절차
- 확률형 아이템 확률 정보 공개
- **성인 인증 기능 필수 연동** (만 19세 이상)

### 결제·정산
- 결제 안전성 확보
- 명확한 환불·취소 정책
- **일·월 결제 한도 관리 시스템 필수**

### 사후 모니터링
- 출시 후 법령 준수, 과몰입 방지 기능, 결제 한도/확률 공개 등 주기적 점검
- 위반 시 콘텐츠 수정 요청 또는 비노출 조치
