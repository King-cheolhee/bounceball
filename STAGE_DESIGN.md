### 2.2 속도 계산 공식

```typescript
// 스테이지 N의 바운스 주기 (초)
function getBouncePeriod(stage: number): number {
  const START = 1.00;  // 스테이지 1
  const END = 0.40;    // 스테이지 20
  const t = (stage - 1) / 19; // 0 ~ 1
  return START - (START - END) * t;
}

// 또는 단순 선형 보간:
// period = 1.00 - 0.03 * (stage - 1)
```

### 2.3 물리 상수 변환

바운스 주기가 짧아지면 물리 엔진의 **중력**과 **바운스 초기 속도**도 함께 조정 필요.

```typescript
// 바운스 주기 T로부터 물리 상수 계산
// (h: 점프 최대 높이, T: 바운스 주기)
//
// h = (g * T²) / 8
// 점프 시 초기 속도 v0 = g * T / 2
//
// 점프 높이를 일정하게 유지하려면:
//   - 짧은 주기 → 빠른 중력 + 빠른 초기 속도

function getPhysicsForStage(stage: number) {
  const period = getBouncePeriod(stage);
  const TARGET_HEIGHT = 300; // px, 점프 최대 높이 고정
  
  const gravity = (8 * TARGET_HEIGHT) / (period * period);
  const bounceVelocity = -gravity * period / 2;
  
  return { gravity, bounceVelocity };
}
```

> 💡 **점프 높이를 일정하게 유지**하는 게 핵심입니다. 그래야 같은 함정 패턴이 스테이지마다 다른 난이도로 느껴짐. 속도만 빨라지는 것.

### 2.4 좌우 이동 속도 조정

공이 바닥에 닿는 빈도가 늘어나면 좌우 이동 거리가 같은 시간에 짧아지는 효과 발생.
→ **MAX_HORIZONTAL_SPEED**는 스테이지마다 약 5%씩 증가하여 보정

```typescript
function getMaxSpeed(stage: number): number {
  const BASE = 300;
  return BASE * (1 + (stage - 1) * 0.05);
}
```

## 3. 체크포인트 시스템

### 3.1 체크포인트 위치

```
스테이지 1  ────────────────────── (시작점)
스테이지 5  ────────────────────── 
스테이지 6  ────────────────────── ⭐ 체크포인트 1
스테이지 10 ────────────────────── 
스테이지 11 ────────────────────── ⭐ 체크포인트 2
스테이지 15 ────────────────────── 
스테이지 16 ────────────────────── ⭐ 체크포인트 3
스테이지 20 ────────────────────── (최종)
```

체크포인트는 **5번째 스테이지를 클리어한 직후 다음 스테이지(6, 11, 16) 진입 시 저장**됩니다.

### 3.2 체크포인트 동작

| 상황 | 처리 |
|----|----|
| **스테이지 5 클리어** | 체크포인트 1 = 6 저장 |
| **스테이지 10 클리어** | 체크포인트 2 = 11 저장 |
| **스테이지 15 클리어** | 체크포인트 3 = 16 저장 |
| **게임오버 (광고 거부)** | 마지막 체크포인트로 돌아감 (처음 1번부터가 아님) |
| **메인메뉴에서 재진입** | 마지막 체크포인트부터 시작 (예: 체크포인트 11이면 11부터) |
| **체크포인트 없음** | 1부터 시작 |

### 3.3 저장 데이터

```typescript
// 앱인토스 Storage SDK 사용
interface ProgressData {
  currentStage: number;        // 현재 진행 중 스테이지
  checkpointStage: number;     // 마지막 체크포인트 (1, 6, 11, 16)
  maxClearedStage: number;     // 최고 클리어 스테이지 (리더보드용)
  totalPlays: number;          // 총 플레이 횟수
  totalDeaths: number;         // 총 사망 횟수
}

// 저장 시점
// 1) 스테이지 클리어 후: currentStage = stage + 1
// 2) 체크포인트 도달 시 (스테이지 5, 10, 15 클리어 후):
//    checkpointStage = 6 / 11 / 16
// 3) maxClearedStage 갱신 시: 리더보드 점수 제출
```

### 3.4 게임오버 시 처리 흐름

```
[목숨 3개 모두 소진]
       ↓
[게임오버 화면]
       ├─→ [광고 보고 부활] → 같은 스테이지 처음부터 (목숨 +3)
       └─→ [메인메뉴로 / 다시시도]
              ↓
       [마지막 체크포인트부터 재시작]
```

**예시**:
- 스테이지 8에서 게임오버 → 광고 거부 → **체크포인트 6**부터 시작
- 스테이지 13에서 게임오버 → 광고 거부 → **체크포인트 11**부터 시작
- 스테이지 3에서 게임오버 → 광고 거부 → **체크포인트 1**(=시작)부터

## 4. 광고 배치 (스테이지 20개 기준 재조정)

이전 "6스테이지부터 5스테이지마다"는 30스테이지 기준이었으므로 20스테이지에 맞게 재설계.

```
[1~5 스테이지]   : 광고 완전히 없음 (튜토리얼 + 첫인상 보호)
[6 스테이지부터]:
  • 스테이지 10, 15 클리어 직후 전면 광고 1회 (총 2회만 노출)
  • 3목숨 소진 시 → 보상형 광고 옵션
```

**변경 이유**: 20스테이지 안에 광고를 너무 자주 띄우면 사용자가 게임을 끝까지 못 마침. 클리어율이 떨어지면 결국 광고 매출도 줄어듦. **체크포인트 진입 직전(=10, 15 클리어 후)에 광고**를 배치해 자연스러운 호흡으로 사용자가 잠시 쉬는 효과.

  type: ElementType;
  x: number;
  y: number;

interface StageData {
  id: number;

  elements: StageElement[];
}

## 7. 난이도 튜닝 가이드

### 7.1 자체 플레이 테스트 기준

각 스테이지는 다음 기준으로 튜닝:

| 챕터 | 1차 클리어율 기대치 | 평균 시도 횟수 |
|----|----|----|
| 1~5 (튜토리얼) | 90%+ | 1~2회 |
| 6~10 (기본) | 70%+ | 2~4회 |
| 11~15 (응용) | 50%+ | 3~6회 |
| 16~20 (마스터) | 30%+ | 5~10회 |

### 7.2 너무 어려운 스테이지 신호

- 1회 시도에 사망 횟수 3회 이상이 일반적
- 클리어해도 "운이 좋았다"는 느낌
- → **함정 사이 안전 구간 늘리거나 위치 조정**

### 7.3 너무 쉬운 스테이지 신호

- 첫 시도에 너무 쉽게 클리어
- 새로운 기술 학습 없음
- → **요소 1개 추가하거나 안전 구간 축소**

### 7.4 속도 곡선 검증

스테이지 10, 15, 20에서 사용자가 "어, 이제 진짜 빠르다"라는 체감을 느껴야 함.
체감되지 않으면 속도 차이 폭을 더 키우는 것 고려 (예: 종착점을 0.35초로).

## 8. 개발 시 우선 작성 순서

스테이지를 작성할 때 다음 순서를 따르면 효율적:

```
1. Stage 1, 5, 10, 15, 20 먼저 작성 (각 챕터 시작과 끝)
   ↓
2. 자체 플레이로 난이도 곡선 큰 그림 확인
   ↓
3. 사이 스테이지 (2,3,4 / 6,7,8,9 / 11~14 / 16~19) 작성
   ↓
4. 전체 통플레이 → 균형 조정
   ↓
5. 외부 테스터에게 1회 플레이 받기 (선택)
```

## 9. 디버그 모드 (개발 편의)

스테이지 튜닝을 위한 개발자 전용 기능:

```typescript
// 개발 모드 한정
if (import.meta.env.DEV) {
  // 키보드 단축키
  window.addEventListener('keydown', (e) => {
    if (e.key >= '0' && e.key <= '9') {
      // 숫자 키로 스테이지 점프
      jumpToStage(parseInt(e.key) * 2 + 1); // 1, 3, 5, 7...
    }
    if (e.key === ']') jumpToNextStage();
    if (e.key === '[') jumpToPrevStage();
    if (e.key === 'g') toggleGodMode(); // 무적
  });
}
```

> ⚠️ 운영 빌드(`npm run build`)에서는 반드시 제거되도록 `import.meta.env.DEV` 가드 필수
