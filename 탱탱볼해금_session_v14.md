> 요지: 콘솔에서 승인된 **전면형(interstitial) 운영 광고 ID**를 코드에 반영했다(`ait.v2.live.b5a8766b61d547e4`). 보상형은 아직 승인 대기라 테스트 ID 유지. **이 변경은 아직 배포 전이며, 현재 심사 중인 버전 `20260711-3`에는 포함돼 있지 않다.**

## 0. 중요 — 심사 중 버전과 master의 광고 상태가 다름

| 대상 | 전면형 광고 ID | 상태 |
|---|---|---|
| 콘솔 심사 중 `20260711-3` | 테스트 ID (`ait-ad-test-interstitial-id`) | 검토 중(화면 잘림 수정만 담김) |
| 로컬 master(커밋 `2bb664f`) | **운영 ID (`ait.v2.live.b5a8766b61d547e4`)** | 미배포 |

→ 다음에 새 버전을 빌드하면 화면 수정 + 운영 전면광고가 함께 들어간다.

## 1. 이번에 한 일

- `app/src/services/ads.ts` `AD_GROUP_ID.interstitial`: 테스트 ID → 운영 ID `ait.v2.live.b5a8766b61d547e4` 교체.
- 보상형(`rewarded`)은 승인 대기 → 테스트 ID(`ait-ad-test-rewarded-id`) 유지.
- 상단/인라인 주석을 현재 광고 ID 상태에 맞게 갱신.
- 검증: `npm run build`(tsc+vite) 통과. 커밋 `2bb664f` push 완료.

## 2. 다음 단계 (사용자 결정 필요)

현재 `20260711-3`(화면 수정)이 **검토 중**이라, 광고 변경을 반영하려면 **새 버전**이 필요하다. 두 갈래:

- **A안(권장·안전): 심사 결과를 기다린다.** `20260711-3` 승인·출시(또는 반려) 후, master(화면 수정+운영 광고) 기준으로 새 `.ait`를 빌드→등록→폰 테스트→검토 요청. 진행 중 요청을 건드리지 않는다.
- **B안: 지금 심사 요청을 취소하고 하나로 묶는다.** `20260711-3` 검토를 취소하고 master로 새 번들을 만들어 화면+광고를 한 번에 재심사. **검토 취소는 사용자 승인 필수** — 임의로 취소하지 않는다.

배포 방법(둘 공통): `app`에서 `npx ait deploy -m "..."` 또는 콘솔 `버전 등록`에 `app/tangtangball.ait` 업로드. 콘솔 메모는 **120자 제한**.

## 3. 주의사항

- **보상형 광고는 여전히 테스트 ID.** 보상형 승인이 나면 같은 방식으로 `ads.ts`에서 교체.
- **샌드박스 테스트 시 운영 전면광고 노출:** 새 버전을 폰(샌드박스)에서 1회 실행할 때 운영 전면광고가 호출된다. 정상 노출/빈 화면 없이 닫히는지 확인. 코드에 onError 폴백은 있음.
- 광고 노출 시점은 게임오버/스테이지 전환 등 기존 로직 그대로 — ID만 바뀜.
- 검토 제출된 버전은 수정 불가. 문제 발견 시 새 번들.
