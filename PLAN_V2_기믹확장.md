## 1. 조사 요약 — 무엇을 배웠나

**조사 대상 (실제 랭킹 근거로 선정한 상위 5개)**

| 게임 | 인기 근거 | 우리에게 주는 것 |
|------|-----------|------------------|
| Red Ball 4 (FDG) | 구글플레이 1억+ 다운로드, 장르 글로벌 1위 | 챕터 커리큘럼·보스 공식·기믹 변주법 |
| 바운스볼 (라온게임즈) | 1천만+ DL, 한국 무료 전체 2위 기록 | **조작이 100% 동일한 직계 벤치마크** — 기믹 백과사전 + 공정성 규칙 |
| Bounce Classic (Genico) | 1천만+ DL (노키아 원작 복각) | 링 수집 해금·타이밍 회랑 |
| Bounce Tales (Rovio/Nokia) | 노키아 프리인스톨 문화 현상 | 런처 적·추격전 피날레 |
| BOUND 원작 공튀기기 (CraftM) | 한국 장르명의 어원, 컴퓨터실 세대 공통 기억 | 화살표 블록·잔상·노미스 칭호 전통 |

**5개 게임 교차 검증으로 확인된 핵심 패턴**
1. **입력은 늘리지 않는다** — 깊이는 발판/지형에 물리를 위임해서 만든다 (신규 기믹 전부 좌/우 입력만으로 성립해야 함)
2. **신규 기믹 커리큘럼**: 안전한 첫 등장 → 기존 요소와 조합 → 챕터 관문에서 종합 시험. 챕터당 신규 2종 이내
3. **위험 = 도구 이중성**: 같은 오브젝트가 함정이자 해법일 때 퍼즐 쾌감 최대 (폭탄이 정확히 이 패턴)
4. **공정성 3원칙** (어기면 평점 테러 — 바운스볼 낙석 사례): ① 스폰 후 3초 무입력 생존 보장 ② 모든 위험은 점멸 예고 ③ 사망 판정은 시각 크기의 80~85%로 후하게
5. **점멸 = 흑백 2색의 만능 신호**: 실선 채움=안전 / 점선=일시적 / 빠른 점멸=위험 예고 / 윤곽선만=허상 — 게임 전체 통일 문법으로 채택

## 2. 신규 기믹 설계

### 필수 3종 (사용자 지정)

#### ① 2회 밟으면 깨지는 벽돌 — `brick`
- **동작**: 1회째 밟으면 균열 무늬 + 짧은 점멸, 2회째 밟으면 파괴. (기존 fragile=1회용의 상위 변주)
- **선례**: Red Ball 4 닌자 박스(2회 히트), 바운스볼 소멸 블록
- **조사 교훈 반영**: 잔여 내구도가 안 보이면 불만 1순위 → **균열 1단계를 반드시 시각화**
- **전용 맵**: 신규 S12 「붕괴 시추공」 — 벽돌 더미 **위에서 시작**, 밟을 때마다 무너지며 **수직 하강**, 벽돌·일반 발판을 갈아타고 하부 탈출구로 (세로 스크롤. S13 자유 낙하가 "빈 공간 낙하+벽타기"라면 S12는 "지형을 부수며 내려가기"로 대비)

#### ② 폭탄으로 벽 파괴 + 넉백 버티기 — `bomb` + `crackedWall`
- **동작**: 공이 폭탄에 닿으면 점화 → **1.2초 빠른 점멸(예고)** → 폭발: (a) 반경 내 '금 간 벽' 파괴 = 길 개방, (b) 공이 폭심 반대 방향으로 강하게 밀려남(수평 임펄스) → **좌/우 입력으로 버티며 착지 제어** — 못 버티면 가시/구덩이로 밀린다
- **선례**: BOUND 폭탄 블록 + CRAZY BOUND '폭탄무효'(폭발을 버티는 수치 — 버티기 콘셉트의 원조), Red Ball 4 붐 박스(폭발 역이용), Bounce Tales 돌 변신(파괴로 길 열기)
- **공정성**: 폭발 반경 점선 원 표시, 넉백 세기·방향 항상 동일, 넉백 후 안전 착지 가능 배치를 시뮬레이션으로 보장, 백업 셀이 실패 완충
- **스토리 스킨**: 과충전 콘덴서 (챕터 3 테마와 정합)

#### ③ 벽 반동 점프(삼각뛰기) 활용 확대
- 신규/개편 맵 전반에 벽 반동 구간 확대: 하강 중 반동 복귀(S12), 넉백 후 반동 복구(S14), 추격전 지름길(S19), 점멸 벽 타이밍 반동(S20)
- **벽 반동으로만 닿는 부품(◆)** 5곳 이상 배치 — 시그니처 기술의 보상화 (Red Ball 4 별 철학)

## 4. 구현 범위 옵션

| 옵션 | 내용 | 신규 기믹 | 재설계 맵 | 예상 작업량 (Claude Code) |
|------|------|----------|----------|---------------------------|
| **A안 — 필수만** | ①②③ | 2종 | 약 6개 (S11·12·14·15·18 + ③확대) | 1세션 ~2시간 |
| **B안 — 필수+저비용** | A + 점멸 발판(P1)·발사 패드(P2)·잔상(P10)·가짜 가시(P11) | 6종 | 약 9개 | 1세션 3~4시간 |
| **C안 — 권장 풀패키지** | B + 이동 가시(P6)·추격전(P9)·완수 메타(P12) | 8종+메타 | 약 12개 | 1~2세션 4~6시간 |

**권장: C안.** 이유 — "맵이 너무 단조롭다"는 피드백의 해법은 챕터마다 다른 플레이 문법(2장=리듬·전류 / 3장=파괴·하강 / 4장=시간·추격)을 주는 것이고, C안이 그 최소 완전체다. 챕터당 신규 2종 이내 원칙도 지켜진다(2장: 점멸+발사 / 3장: 벽돌+폭탄 / 4장: 이동가시+추격전). 잔상·가짜가시·완수 메타는 코드 반나절급 저비용 항목이다.

## 5. 기술 설계 개요 (개발자용 메모)

- **타입**: floor variant에 `brick`(내구도 2)·`blink`(주기·위상) 추가, 신규 StageElement `bomb`·`cracked_wall`·`launcher`·`moving_spike`, spike에 `fake` 플래그
- **물리**: 수평 임펄스 모듈 1개 신설 (폭발 넉백·발사 패드 공유) — 기존 `OVERSPEED_DECAY` 감쇠 재사용, 반대키 제동은 기존 `BRAKE_MULTIPLIER` 경로
- **충돌**: 점멸 발판 비활성 시 통과(스윕 충돌 분기), 이동 가시는 결정적 경로의 동적 AABB
- **렌더**: 모두 Canvas 절차 드로잉, 흑백 2색 + 점멸 문법 (외부 에셋 0개 유지)
- **저장**: 완수 메타 플래그(gameStore), 잔상 토글(설정)

## 6. 검증 계획 (기존 방법론 동일)

1. `tsc --noEmit` + 빌드 + 헤드리스 실플레이(`verify-browser.cjs` — 신규 기믹 스테이지 봇 클리어 추가)
2. 물리 시뮬레이션: `validate2.cjs`·`climb-sim.cjs` 확장 + **신규 `knockback-sim.cjs`**(폭탄 넉백 전수: 모든 폭탄 배치에서 버티기 성공 시 안전 착지 보장·실패 시 사망 지점 의도 확인), 벽돌 하강맵 경로 도달성, 점멸 주기-바운스 주기 정합
3. 멀티에이전트 적대적 리뷰 (리뷰어→회의적 검증자 교차)
4. 휴대폰 실기기 손맛 확인 (사용자) — 특히 넉백 버티기 강도, 점멸 주기 체감

## 7. 작업 순서 (승인 후)

1. 물리·타입 기반 작업 (임펄스 모듈, 신규 요소 타입) → 커밋
2. 기믹 단위 구현+시뮬레이션 검증 → 기믹당 커밋 (벽돌 → 폭탄 → 점멸 → 발사 → 이동가시 → 잔상·가짜가시·메타)
3. 스테이지 개편 (챕터 단위 커밋, 매번 전수 시뮬레이션)
4. 추격전·종합 검증·적대적 리뷰 → 최종 커밋
