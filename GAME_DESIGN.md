# GAME_DESIGN.md

> **탱탱볼해금 (Tangtangball)** — 게임 디자인 문서
> 본 문서는 게임의 컨셉, 메커니즘, 디자인 톤을 정의합니다.

---

## 1. 게임 개요

| 항목 | 내용 |
|----|----|
| **게임 이름** | 탱탱볼해금 (Tangtangball) |
| **장르** | 퍼즐/캐주얼 (타이밍 액션) |
| **플랫폼** | 토스 앱인토스 미니앱 |
| **타겟 사용자** | 만 19세 이상 토스 사용자 (시간 때우기형 캐주얼 게이머) |
| **영감** | BOUND (플래시 게임 "원조 공튀기기"), 클래식 LCD 게임 |
| **핵심 가치 제안** | 복잡한 사고 없이, 순간적 타이밍에 집중하는 5분 짜투리 시간용 게임 |

### 1.1 한 줄 컨셉

> 좌우로 자동 튕기는 공의 방향만 제어해 함정을 피하고 스테이지를 통과하는 미니멀 흑백 LCD 스타일 타이밍 퍼즐 게임

### 1.2 핵심 재미 (Core Fun)

- **단순함의 깊이**: 좌우 두 개의 버튼만으로 조작하지만, 관성과 다양한 바닥 타입이 더해져 깊이가 생김
- **즉각적 피드백**: 한 번의 잘못된 타이밍이 즉시 결과로 이어지는 명확함
- **레트로 향수**: 흑백 LCD 게임의 향수를 자극하는 비주얼/사운드

## 2. 게임 메커니즘

### 2.1 코어 루프

```
1. 공이 자동으로 위아래로 튕긴다 (중력 + 반발력)
2. 플레이어가 화면 좌/우 영역을 터치하면, 공이 해당 방향으로 가속
3. 손을 떼면 관성에 의해 속도가 점차 줄어들면서 제자리 바운스로 전환
4. 공이 좌→우로 전진하며, 화면은 공을 따라 축 스크롤
5. 함정(구멍, 가시, 폭발 발판 등)을 피해 스테이지 끝까지 도달하면 클리어
6. 함정에 빠지면 목숨 1개 소진, 같은 스테이지 재시작
7. 3목숨 모두 소진 시 → 광고 부활 옵션 / 스테이지 처음부터
```

### 2.2 공의 물리 시스템

#### 자동 바운싱
- 공은 항상 위아래로 자동 튕긴다 (수직 운동)
- 바운싱 높이/주기는 일정 (개발 시 조정 가능한 상수)

#### 수동 좌우 제어 (관성 기반)
- **좌측 터치**: 공이 왼쪽으로 가속
- **우측 터치**: 공이 오른쪽으로 가속
- **터치 떼기**: 관성으로 속도가 점차 감속 → 결국 제자리 바운스로 전환
- **반전**: 진행 중 반대 방향 터치 시, 점진적 감속 후 반대 방향 가속

#### 벽 충돌
- 좌/우 벽에 부딪히면 공은 반대 방향으로 튕긴다 (자연스러운 반사)
- 벽 충돌 순간 반대 방향키를 누르면, 더 빠르게 튕겨나갈 수 있음

### 2.3 입력 방식

- **모바일 가로 모드** 전용
- **화면 좌/우 반반 분할 터치**:
  - 화면 왼쪽 절반 터치 = 좌측 이동
  - 화면 오른쪽 절반 터치 = 우측 이동
- **멀티 터치 지원**: 좌/우 동시 터치 시 후속 입력 우선
- **터치 떨어지면 입력 종료** → 관성 시작

## 3. 스테이지 시스템

### 3.1 진행 구조

- **총 20 스테이지** (1 ~ 20)
- **순차 진행형**: 1 → 2 → 3 → ... 순서로 자동 진행
- **스테이지 선택 화면 없음**: 현재 진행 중인 스테이지로 바로 입장
- **클리어 시**: 다음 스테이지 자동 진입
- **클리어 평가**: 단순 클리어/실패만 기록 (별 평가 시스템 없음)

### 3.2 체크포인트 시스템 ⭐

- **5스테이지 단위 자동 저장**: 스테이지 5, 10, 15 클리어 시 다음 체크포인트(6, 11, 16) 저장
- **체크포인트 위치**: 1, 6, 11, 16
- **게임오버 또는 재진입 시**: 마지막 체크포인트부터 시작 (처음 1부터가 아님)
- **예시**: 스테이지 8에서 게임오버 → 체크포인트 6부터 재시작

> 자세한 체크포인트 동작은 `STAGE_DESIGN.md` 3장 참조

### 3.3 목숨 시스템

- **스테이지당 3목숨 기본 제공**
- 함정에 빠지면 → 목숨 1 차감 → 해당 스테이지 처음부터 재시작
- **3목숨 모두 소진 시**: 게임오버 화면 표시
  - 옵션 A: **광고 보고 부활** (보상형 광고) → 목숨 +3 추가, 스테이지 처음부터
  - 옵션 B: **포기하고 메인 메뉴로** → 마지막 체크포인트부터 시작

### 3.4 스테이지 구성 요소

#### 바닥 타입
| 종류 | 설명 | 시각 표현 |
|----|----|----|
| **일반 바닥** | 무한 사용 가능, 공이 안정적으로 튕김 | 솔리드 흰색 라인 |
| **구멍** | 바닥이 없는 구간, 빠지면 즉시 사망 | 빈 공간 (검정 배경 노출) |
| **부서지는 바닥** | 1회 밟으면 사라짐 (다음 바운스 시 빠짐) | 점선 흰색 라인 |
| **폭발 발판** | 밟는 순간 폭발 → 즉시 사망 | 빗금 패턴 또는 깜빡임 |

#### 위협 요소
| 종류 | 설명 | 시각 표현 |
|----|----|----|
| **가시** | 위에서 공이 닿으면 즉시 사망 | 위로 향한 삼각형 |
| **천장 가시** | 공이 위로 튀어 닿으면 사망 (14스테이지부터 등장) | 아래로 향한 삼각형 |
| **옆 벽** | 공이 부딪히면 반사 (사망 아님) | 수직 흰색 라인 |

### 3.5 난이도 & 속도 곡선

| 챕터 | 스테이지 | 테마 | 바운스 주기 |
|----|----|----|----|
| **튜토리얼** | 1~5 | 조작 익히기 (구멍/가시) | 1.00초 → 0.88초 |
| **기본** | 6~10 | 부서지는 바닥 도입 | 0.85초 → 0.73초 |
| **응용** | 11~15 | 폭발 발판 + 복합 함정 | 0.70초 → 0.58초 |
| **마스터** | 16~20 | 정교한 타이밍 + 극한 속도 | 0.54초 → 0.40초 |

- **속도 점진 증가**: 스테이지 1(1초/바운스) → 스테이지 20(0.4초/바운스)
- **MVP 스테이지 수**: **20 스테이지**

> 💡 스테이지별 상세 설계, 속도 곡선 공식, 모든 함정 배치는 `STAGE_DESIGN.md` 참조

## 4. 디자인 톤 & 비주얼

### 4.1 아트 디렉션

- **컨셉**: 모노크롬 클래식 아케이드 (Monochrome Classic Arcade)
- **레퍼런스**: 게임보이 오리지널, 닌텐도 Game & Watch, 초기 LCD 게임

### 4.2 컬러 팔레트

```
배경    : #000000 (Pure Black)
오브젝트 : #FFFFFF (Pure White)
강조    : #FFFFFF (동일, 굵기/크기로 차별)
```

- **단 두 가지 색만 사용** (스테이지 변경에도 동일)
- **계조 표현 없음**: 알파/그라데이션 사용 금지 (LCD 느낌 보존)
- 단, 게임오버/일시정지 시 흰색을 50% 알파로 처리하는 정도는 허용 (가독성)

### 4.3 형태 디자인

- **공**: 완전한 원형 (perfect circle)
- **바닥**: 굵은 흰색 수평 라인 (2~4px 두께)
- **가시**: 위로 뾰족한 삼각형
- **벽**: 굵은 흰색 수직 라인
- **모든 요소는 안티에일리어싱 처리** (게임 자체는 픽셀 아트 아님, 매끄러운 도형)

### 4.4 타이포그래피

- **컨셉**: 세련된 모던 폰트 (레트로 비주얼과 대조)
- **추천 폰트**:
  - **Pretendard** (한글)
  - **Inter** 또는 **Space Grotesk** (영문/숫자)
- **사용처**:
  - 점수, 스테이지 번호, 타이머
  - 메뉴 텍스트, 버튼 라벨

### 4.5 모션 디자인

- **공 잔상**: 빠른 이동 시 짧은 모션 트레일 (1~2 프레임)
- **충돌 피드백**: 벽/바닥 충돌 시 0.05초 화면 미세 진동 (선택적)
- **사망 연출**: 공이 함정에 닿는 순간 화면 깜빡임 (검정 ↔ 흰색 1회)
- **스테이지 전환**: 페이드 인/아웃 (검정 → 게임 → 검정)

## 6. 화면 구조

### 6.1 화면 흐름도

```
┌────────────┐
│  스플래시  │ (1~2초)
└─────┬──────┘
      ↓
┌────────────┐
│ 메인 메뉴  │ ─── [설정 버튼] ─→ [설정 화면]
└─────┬──────┘                    │
      │                            └──→ (뒤로) ──┐
      │ [시작 버튼]                              │
      ↓                                          ↓
┌────────────┐                            ┌────────────┐
│ 게임 플레이│                            │ 메인 메뉴  │
└─────┬──────┘                            └────────────┘
      │
      ├──[일시정지]──→ 계속/다시하기/메인메뉴
      │
      ├──[클리어]──→ 다음 스테이지 (자동)
      │            └──[10, 15 클리어 직후]──→ 전면 광고 → 다음 스테이지
      │
      └──[3목숨 소진]──→ ┌─────────────┐
                          │ 게임오버    │
                          └─────────────┘
                            ├─ [광고 보고 부활] (보상형) → 스테이지 처음부터 (목숨 +3)
                            ├─ [다시 시도]              → 마지막 체크포인트부터
                            └─ [메인 메뉴로]            → 메인 메뉴 (다음 진입 시 체크포인트부터)
```

### 6.2 화면별 구성 요소

#### 6.2.1 스플래시 화면
- 로고 (게임 이름 "탱탱볼해금" 또는 심볼) 중앙 배치
- 1~2초 후 자동 메인 메뉴로 전환
- **로딩 시간 10초 이내 필수** (앱인토스 검수 요구사항)

#### 6.2.2 메인 메뉴
- **게임 타이틀** (상단 중앙)
- **현재 진행 중 스테이지 표시** (예: "Stage 7")
- **시작 버튼** (중앙, 큰 크기)
- **리더보드 버튼** (하단 좌측)
- **설정 버튼** (하단 우측)

#### 6.2.3 게임 플레이 화면
- **상단 좌측**: 현재 스테이지 번호
- **상단 우측**: 남은 목숨 (♥ 3개 표시)
- **상단 중앙**: 일시정지 버튼 (II)
- **메인 영역**: 게임 캔버스 (가로 스크롤)
- **터치 영역**: 화면 좌/우 반반 분할 (시각적 표시 없음, 투명)

#### 6.2.4 일시정지 오버레이
- 배경 50% 어둡게 처리
- **계속하기** / **다시하기** / **메인 메뉴** 3개 버튼

#### 6.2.5 게임오버 화면
- "GAME OVER" 텍스트
- **[광고 보고 부활하기]** (보상형 광고 트리거)
- **[다시 시도]**
- **[메인 메뉴]**

#### 6.2.6 클리어 트랜지션
- "STAGE 7 CLEAR" 텍스트 1초간 표시
- 자동 다음 스테이지 진입

#### 6.2.7 설정 화면
- **사운드 ON/OFF** 토글
- **진동(햅틱) ON/OFF** 토글
- **(선택) 진행 데이터 초기화** 버튼

#### 6.2.8 리더보드 (앱인토스 게임 센터)
- 앱인토스 SDK의 리더보드 화면 호출 (자체 구현 불필요)
- 정렬 기준: 최고 클리어 스테이지

## 7. 수익화 전략 (인앱 광고)

### 7.1 전체 광고 배치

```
[1~9 스테이지]   : 광고 완전히 없음 (튜토리얼 + 첫인상 보호)
[10, 15 스테이지 클리어 직후]:
  • 전면 광고 1회 (체크포인트 진입 직전 자연스러운 호흡)
  • 총 2회만 노출 (20스테이지 게임 분량에 맞게 절제)
[3목숨 소진 시 (전 스테이지 공통)]:
  • 보상형 광고 옵션 (사용자 자발 시청 → 목숨 +3, 같은 스테이지 처음부터)
```

### 7.2 전면형 광고 (Interstitial)

- **노출 시점**: 스테이지 10, 15 클리어 직후 (총 2회)
- **사전 로딩 필수**: 광고 재생 시점에 실시간 로딩 금지
- **재생 중**: 게임 음악 일시정지
- **종료 후**: 다음 스테이지(11, 16 = 체크포인트) 진입 + 음악 재생
- **설계 의도**: 새 챕터 시작 전 자연스러운 휴식 + 체크포인트 직전이라 거부감 적음

### 7.3 보상형 광고 (Rewarded)

- **노출 시점**: 게임오버 화면 (3목숨 모두 소진 시)
- **시청 보상**: 목숨 3개 추가 → 같은 스테이지 처음부터 재시작
- **사용자 선택**: 거부 시 일반 재시작 또는 메인메뉴 이동
- **시청 완료 검증 필수**: 광고를 끝까지 봐야 보상 지급

### 7.4 광고 정책 안전장치

- ❌ 인트로/로딩/팝업 모달에 광고 노출 금지 (앱인토스 검수 반려 사유)
- ❌ 예상치 못한 순간의 광고 노출 금지
- ✅ 광고 재생 중 BGM/SFX 자동 일시정지
- ✅ 광고 종료 후 자동 화면 복귀
- ✅ 보상형 광고는 사용자 선택에 의해서만 트리거

| 키 | 데이터 | 설명 |
|----|----|----|
| `current_stage` | number | 현재 진행 중 스테이지 (다음 진입 시 복원) |

### 10.1 토스 정책 준수
